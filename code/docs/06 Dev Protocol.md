# Dev protocol
1. Divide complex goals into subgoals.
2. Make a new directory and start a new thread for a each subgoal.
3. Create logs of each subgoals and goals as markdown files to log attempts and results step by step with reasoning.
4. Summarize attempts and results for each subgoal in jupyter notebook format, utilizing plots as much as possible, but only for critical information.
5. Save codes in /root/capsule/code/dev_code, notebooks in /root/capsule/code/notebooks.
6. Number each notebook to show progression.
7. Organize relevant files in /root/capsule/scratch into folders.

## Rule
- Do not use benchmark data information for developing automatic registration.
- Use benchmark dataset for understanding the problem conditions and constraints.
- Use benchmark dataset for validation after the development.
